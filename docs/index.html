BasicPlus

Un lenguaje de propósito general para microcontroladores de 32 bits: sintaxis BASIC moderna, orientación a objetos y un depurador de primera clase. Compila a bytecode que corre idéntico en el PC y en el micro.

V2 · código congelado RP2350 · ESP32-S3 · STM32 Dos VMs · una salida byte-idéntica Probado en hardware real MIT

El mismo programa que depuras en tu sobremesa parpadea después un LED en una Raspberry Pi Pico 2, un ESP32-S3 o un STM32 — sin recompilar, sin #ifdef, sin sorpresas. En un PC sin GPIO los builtins de hardware loggean por la consola; en la placa mueven pines de verdad. El bytecode es el mismo byte a byte. 

// blink.bp — parpadea el LED on-board de la Pico (GP25) con el API OO.
module Blink
  import Gpio

  function Main()
    var led: Gpio.Pin := Gpio.Pin(25, Gpio.Pin.OUTPUT)
    var i: integer := 0
    while i < 100 do
      led.on()
      sleep(300)
      led.off()
      sleep(300)
      i := i + 1
    endwh
    print "Blink terminado"
  end Main
end Blink

Verificado en hardware real

No es solo teoría de sobremesa: la pila completa se ha ejercitado en silicio. En una Raspberry Pi Pico 2 / Pico 2 W (RP2350) se validó de punta a punta —desde el IDE, con Run on Device y el mismo bytecode que corre en el PC— toda la escalera de hardware:

SubsistemaPrueba en placa
Ejecución / OOclases stdlib cross-module despachando en el micro ✅
GPIOLED parpadeando (Gpio.Pin) ✅
I2Csensor real BMP280 (T/P) + escaneo de bus ✅
SPIBME688 por SPI: chip_id + lecturas T/HR/P/gas con paginación de memoria ✅
UARTloopback, eco de bytes ✅
PWM + contador1 kHz / 500 Hz contados por hardware, < 0.1 % de error ✅
ADCtemperatura interna del chip (23.9 °C) ✅
RTCreloj monotónico + recalibración ✅
Watchdogfeed / timeout / disable ✅
Timersalarmas hardware (polling, sincronizado, cronómetro) ✅

En el STM32 (Nucleo-U575ZI-Q) se validaron además en placa, con sensores reales, los cuatro buses críticos: GPIO, SPI (BME688), UART (loopback) e I2C (BME280, T/P); los periféricos no críticos (PWM/ADC/RTC/WDT) existen en la API y se ejecutan, pero su backend en STM32 queda para v3. Metro RP2350B y ESP32-S3 comparten el mismo núcleo de VM y arrancan, ejecutan y se comunican con el IDE; la pasada exhaustiva de periféricos por placa está en curso.

El nombre. BasicPlus es un homenaje al BASIC-PLUS de Digital Equipment Corporation, el dialecto estructurado que corría bajo RSTS/E en los PDP-11 en los años 70 y 80 — aquel BASIC con el que se escribieron sistemas completos y aprendió a programar una generación. La misma idea (un lenguaje sencillo de leer que no te trata como a un principiante) trasplantada al hardware pequeño de hoy.
